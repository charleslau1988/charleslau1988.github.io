▶ 一、项目背景与量化目标SMART 8项
▶ 1.1 设备制造数据环境画像
产品复杂度：3000-50000 个零部件/台，多层 BOM 制造模式：多品种小批量 + ETO + 现场定制 核心系统：PLM / ERP / MES / QMS / SCM / CRM / IoT 日均数据量：10-100 GB（设计文件/装配记录/测试报告） 数据层级：整机SN → 模组SN → 部件批次 → 物料编码 生命周期：10-20 年（出厂到退役全程追溯）

▶ 1.4 数据治理价值与可分析问题
装配质量追溯：8小时→15分钟 库存呆滞降低：消除一物多码 维修响应加速：MTTR数据即时查询 跨部门对账：3-5天→1天 停线返工减少50-60%
质量问题分析场景：
├─ 装配不良集中在哪个模组/工序 → fact_Assembly 维度聚合
├─ 同一零件不同批次质量波动 → dim_Part + fact_Inspection 趋势
├─ BOM版本切换前后质量对比 → dim_BOM SCD Type 2 回溯
├─ 老化通过率与温箱相关性 → fact_TestResult + BurnInRoomID
├─ 供应商来料不良排名 → fact_Inspection 按 Supplier 聚合
├─ 故障代码分布 → fact_Maintain + FaultCode 交叉分析

运营效率分析：Cycle Time趋势 / Top N失效零件 / 瓶颈工序 / 备件消耗预测
合规风控：出口管制零件流向 / 设计图纸访问审计 / 客户数据隔离

▶ 四、核心数据模型：设备 SN + BOM 双 Hub10张核心表
▶ 4.1 双 Hub 建模逻辑
SN Hub — 设备一生一本账：每台设备出厂SN唯一编码，贯穿装配→检验→出货→装机→维保→退役

BOM Hub — 版本化管理：一台设备经历多次升级后BOM结构完全不同，需要独立版本追踪

ID Mapping：SN ↔ MES资产号 ↔ ERP固定资产号 ↔ CRM设备号 ↔ 客户设备标签
内部物料号 ↔ 供应商料号 ↔ 客户指定料号
▶ 4.2 核心维表设计
dim_Equipment（设备维度，SN Hub）：
EqpSN(UQ) · EqpModel · EqpCategory · CustName · ShipDate · InstallDate
CurrentBOMVersion · OriginalBOMVersion · EqpStatus · SecurityLevel
SCD字段：ValidFrom · ValidTo · IsActive

dim_BOM（BOM维度，BOM Hub）：
EqpModel + BOMVersion(UQ) · BOMStatus(Draft/Released/Obsolete) · CustCode

dim_BOMDetail（BOM明细，多层展平）：
BOMID(FK) · ParentPartID · PartID · PartQty · LevelNo(0=整机/1=模组/2=子模组) · Position · IsCritical · ReplaceablePartNo

PartNo(UQ) · PartName · PartCategory(机械件/电气件/耗材/标准件) · SupplierID

▶ 4.3 核心事实表设计

fact_TestResult（测试结果）：TestType(出厂/老化/校准/现场复测) · TestItem · TestValue · SpecLower · SpecUpper · Result

▶ 5.1 四层标准架构

▶ 5.2 物料编码规则
XX - YY - ZZZ - NNNNN
├─ XX = 大类码（RC=电阻, CC=电容, IC=芯片, ME=金属件, SE=密封件）
├─ YY = 中类码（01=贴片, 02=插件, 03=定制加工件）
├─ ZZZ = 小类码（001=0402封装, 002=0603封装）
└─ NNNNN = 流水号（5位数字）
▶ 5.3 装配/测试参数命名规范
命名规则：{工序域}_{参数名}_{单位}

⚙️ 机械装配 (MECH)：MECH_TORQUE_Nm · MECH_ALIGNMENT_um · MECH_GAP_mm

▶ 7.2 ETL 五步流水线（装配工单）
Step 1：Raw 层 — 零清洗采集
INSERT INTO raw_mes_assembly SELECT * FROM source，只加批次号和采集时间戳

Step 2：异常检测 — 脏数据隔离
工序码不在标准字典 → INSERT INTO exc_mes_assembly（不阻断流程）

Step 3：Staging 层 — 字段标准化 + ID 映射
LEFT JOIN map_EquipmentIdentity 解析 UnifiedEqpID，工序码转标准名称

Step 4：DW 层 — MERGE 幂等写入
MERGE INTO fact_Assembly ON WorkOrderNo+ProcessStep+StartTime 做幂等键

Step 5：行数对账 — 确保数据完整
SELECT Raw, Stg, Exc, DW → 预期 Raw = Stg + Exc = DW，不等则告警暂停
▶ 7.3 装配与测试数据三层治理
采集层：量具/仪器校准 + 漂移检测 | 缺失值策略(未测/仪器异常) | 采集规范
分析层：规格限/公差限/统计控制限 | 告警分级：Critical(停线) / Warning(24h) / Monitor(趋势) | 规则版本化+Owner
闭环层：不合格品→根源分析→纠正措施→验证闭环 | 知识库积累
▶ 7.4 BOM一致性治理
EBOM→MBOM转换：设计BOM注入辅料/耗材，分配工位 一致性比对：PLM EBOM vs MES MBOM 物料/用量/差异分析 变更联动：ECN触发影响评估（在产工单+在用设备升级）
